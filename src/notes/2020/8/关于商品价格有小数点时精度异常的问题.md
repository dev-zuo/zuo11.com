# 关于商品价格有小数点时精度异常的问题

在价格显示时，如果有小数位，**由于js浮点数的缺陷，累加的价格可能会有经度差，我们直接 toFixed(2) 即可**。一般不会超过3位小数点，且精度异常一般都是小数点很多位以后，注意js小数点运算时一定要注意这个问题，确定要保留几位小数点

```js
37998.91 + 0.01
// 37998.920000000006

0.1+0.2
0.30000000000000004
```
